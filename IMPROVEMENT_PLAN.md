## 🎯 1. 사용성 (UX/UI) 개선 사항

### 🔴 긴급 (Critical)

#### 1. 결제 실패/취소 시 주문 데이터 처리 누락
- **현재 상황**: Toss 결제 취소 시 `pending` 상태 주문이 DB에 계속 남음
- **문제점**: 관리자가 수동으로 구분해야 하고, 통계가 왜곡됨
- **개선 방안**: 
  - 결제 실패 시 주문 상태를 `cancelled` 또는 `failed`로 자동 업데이트
  - `/order/fail` 페이지에서 API 호출하여 주문 상태 변경
  - 관리자 대시보드에서 실패한 주문 필터링 옵션 추가

#### 2. 주문 조회 시스템 부재
- **현재 상황**: 주문 후 주문번호나 조회 수단이 없음
- **문제점**: 고객이 본인 주문 상태를 확인할 수 없음
- **개선 방안**:
  - 주문 완료 후 주문번호를 SMS/이메일로 발송
  - 비회원 주문 조회 페이지 생성 (`/order/track`)
    - 입력: 주문번호 + 전화번호 또는 이메일
    - 출력: 주문 상세 정보, 배송 상태
  - 회원은 마이페이지에 주문 내역 탭 추가 (`/mypage/orders`)
  - 주문 상태별 알림 (주문접수 → 결제완료 → 상품구매 → 배송중 → 배송완료)

#### 3. 검색 기능 미구현
- **현재 상황**: 헤더에 검색바가 있지만 실제 기능이 없음
- **문제점**: 사용자가 특정 상품을 빠르게 찾을 수 없음
- **개선 방안**:
  - `/search` 페이지 구현
  - 상품명, 브랜드, 카테고리 기반 검색
  - 실시간 자동완성 (debounce 적용)
  - 검색 결과 필터링 (가격대, 카테고리, 정렬)
  - 최근 검색어 저장 (localStorage)
  - 인기 검색어 표시

### 🟠 중요 (High Priority)

#### 4. 상품 재고/품절 관리 없음
- **현재 상황**: 모든 상품이 항상 구매 가능한 것처럼 표시
- **문제점**: 품절 상품 주문 시 환불/취소 처리 부담
- **개선 방안**:
  - `products` 테이블에 `stock_quantity`, `is_in_stock` 컬럼 추가
  - 품절 시 "품절" 배지 표시 및 구매 버튼 비활성화
  - 재입고 알림 신청 기능
  - 관리자 페이지에서 재고 관리 기능

#### 5. 배송 추적 시스템 부재
- **현재 상황**: 주문 상태만 표시 (pending, paid, shipped, delivered)
- **문제점**: 고객이 정확한 배송 위치를 모름
- **개선 방안**:
  - `orders` 테이블에 `tracking_number`, `carrier` 컬럼 추가
  - 배송 단계별 상세 안내
    1. 주문접수 (Order Received)
    2. 일본 현지 구매 중 (Purchasing in Japan)
    3. 한국 배송 중 (Shipping to Korea)
    4. 통관 중 (Customs Clearance)
    5. 배송완료 (Delivered)
  - 운송장 번호로 외부 택배사 추적 링크 연동
  - 단계별 예상 소요 시간 표시

#### 6. 가격 변동 알림 없음
- **현재 상황**: 환율/세금이 변경되면 가격이 바뀌지만 알림 없음
- **문제점**: 사용자가 찜한 상품의 가격 변화를 놓침
- **개선 방안**:
  - 찜한 상품의 가격을 DB에 저장
  - 매일 크론잡으로 가격 변동 체크
  - 가격 하락 시 이메일/앱 푸시 알림
  - 마이페이지에서 "가격 변동 알림 설정" 토글

#### 7. 모바일 UX 개선 필요
- **현재 상황**: 주문서 페이지에서 스크롤이 많이 필요함
- **문제점**: 모바일에서 주문 과정이 번거로움
- **개선 방안**:
  - 아코디언 형식으로 섹션 접기/펼치기
  - 배송지 정보는 요약만 표시하고 수정 시에만 펼치기
  - 스텝 인디케이터 추가 (1.상품확인 → 2.배송지 → 3.결제)
  - 하단 고정바의 높이를 줄여 콘텐츠 가시성 향상

#### 8. 에러 메시지 개선 필요
- **현재 상황**: "저장 중 오류가 발생했습니다" 같은 일반적인 메시지
- **문제점**: 사용자가 무엇이 문제인지 모름
- **개선 방안**:
  - 구체적인 에러 메시지
    - "전화번호 형식이 올바르지 않습니다 (예: 010-1234-5678)"
    - "비밀번호는 최소 8자 이상, 영문+숫자 조합이어야 합니다"
    - "이미 가입된 이메일입니다"
  - 입력 필드에 실시간 유효성 검사
  - 에러 발생 시 해당 필드로 스크롤 이동

---

### 🟡 권장 (Medium Priority)

#### 9. 상품 비교 기능 없음
- **개선 방안**:
  - 상품 카드에 "비교하기" 체크박스 추가
  - 최대 3-4개 상품 선택
  - 비교 페이지에서 가격, 스펙, 배송비 등을 테이블 형식으로 표시
  - 가장 저렴한 항목 하이라이트

#### 10. 상품 상세 이미지가 1개만
- **현재 상황**: 썸네일 1개만 표시
- **개선 방안**:
  - `products` 테이블의 `image_url`을 배열로 변경 또는 별도 테이블
  - 이미지 갤러리 UI (슬라이더)
  - 확대/축소 기능 (Zoom In)
  - 관리자 페이지에서 다중 이미지 업로드

#### 11. 쿠폰/프로모션 시스템 없음
- **개선 방안**:
  - `coupons` 테이블 생성
    - 쿠폰 코드, 할인율/금액, 유효기간, 사용 조건
  - 결제 시 쿠폰 입력란 추가
  - 자동 적용 쿠폰 (첫 구매 10% 할인)
  - 마이페이지에서 보유 쿠폰 확인

#### 12. 최근 본 상품 기능 없음
- **개선 방안**:
  - localStorage에 최근 본 상품 ID 저장
  - 메인 페이지 또는 사이드바에 "최근 본 상품" 섹션
  - 최대 10개까지 저장
  - 비회원도 가능

## 💰 2. 사업성 (Business) 개선 사항

### 🔴 긴급 (Critical)

#### 13. 수익성 분석 도구 부재
- **현재 상황**: 관리자 대시보드에 총 매출만 표시
- **문제점**: 실제 수익률(마진)을 모름
- **개선 방안**:
  - 상품별 수익 계산
    - 실제 일본 구매가 (관리자가 입력)
    - 실제 배송비 (관리자가 입력)
    - 순수익 = 판매가 - (구매가 + 배송비)
  - 대시보드에 추가 지표
    - 월별/일별 수익 트렌드 그래프 (Chart.js)
    - 평균 마진율
    - 마진율이 낮은 상품 TOP 10 (자동 알림)
    - 가장 많이 팔린 상품 TOP 10
  - 엑셀 다운로드 기능 (월별 정산 보고서)

#### 14. 고객 이탈 방지 장치 없음
- **현재 상황**: 장바구니에 담고 나가면 끝
- **문제점**: 구매 의사가 있는 고객을 놓침
- **개선 방안**:
  - 장바구니 기능 추가
    - 여러 상품을 담아서 한 번에 결제
    - 장바구니 페이지 (`/cart`)
  - 장바구니 알림 (24시간 후)
    - "장바구니에 상품이 있습니다. 지금 구매하시겠어요?"
  - 이메일 리마케팅
    - 찜한 상품 가격 하락 알림
    - 회원가입 후 첫 구매 유도 (웰컴 쿠폰)
    - 장기 미접속 고객 재방문 유도

#### 15. 결제 수단이 카드만 가능
- **현재 상황**: Toss Payments 카드 결제만
- **문제점**: 무통장입금, 가상계좌, 간편결제를 선호하는 고객 이탈
- **개선 방안**:
  - Toss Payments에서 제공하는 다양한 결제 수단 추가
    - 카드
    - 계좌이체
    - 가상계좌
    - 휴대폰 소액결제
    - 카카오페이, 네이버페이, 토스페이
  - 결제 수단 선택 UI 개선

### 🟠 중요 (High Priority)

#### 16. 리텐션 전략 부족
- **현재 상황**: 회원 가입 혜택이 없음
- **문제점**: 재구매율이 낮을 수 있음
- **개선 방안**:
  - **회원 등급제**
    - 일반 → 실버 → 골드 → VIP
    - 등급별 혜택 (추가 할인, 무료 배송 쿠폰, 적립금 추가 지급)
    - 등급 승급 조건 (주문 횟수, 누적 구매 금액)
  - **포인트/적립금 시스템**
    - 현재 "결제 금액의 1% 적립 예정"이라고 표시만 하지만 실제로 적립 안 됨
    - `points` 테이블 생성 (적립/사용 내역)
    - 결제 시 포인트 사용 옵션
    - 리뷰 작성 시 추가 포인트 지급
  - **생일 쿠폰**
    - 회원가입 시 생년월일 입력
    - 생일 달에 특별 할인 쿠폰 발송
  - **추천인 제도**
    - 친구 초대 시 양쪽 모두에게 쿠폰 지급

#### 17. 마케팅 데이터 수집 부족
- **현재 상황**: 회원가입 시 "마케팅 수신 동의"만 받음
- **문제점**: 실제 마케팅 효과를 측정할 수 없음
- **개선 방안**:
  - **Google Analytics 4 설치**
    - 페이지뷰, 이벤트 추적
    - 전환율 분석
    - 이탈률 높은 페이지 파악
  - **Meta Pixel (Facebook/Instagram) 설치**
    - 리타게팅 광고 가능
    - 유사 타겟 고객 찾기
  - **UTM 파라미터 추적**
    - 광고 캠페인별 유입 경로 분석
    - 가장 효과적인 마케팅 채널 파악
  - **히트맵 도구** (Hotjar, Microsoft Clarity)
    - 사용자가 어디를 클릭하는지 시각화
    - 스크롤 깊이 분석

#### 18. 관리자 대시보드 통계 부족
- **현재 상황**: 총 상품/주문/매출만 표시
- **문제점**: 의사결정에 필요한 데이터가 부족함
- **개선 방안**:
  - **추가 지표**
    - 일별/주별/월별 방문자 수 (GA 연동)
    - 전환율 (방문자 대비 주문)
    - 평균 주문 금액 (AOV - Average Order Value)
    - 재구매율 (지난 30일 내 재주문한 고객 비율)
    - 카테고리별 매출 비중 (파이 차트)
    - 시간대별 주문 현황 (히트맵)
  - **그래프 시각화** (Chart.js, Recharts)
    - 월별 매출 추이 (라인 차트)
    - 카테고리별 매출 (바 차트)
    - 주문 상태별 분포 (도넛 차트)
  - **실시간 알림**
    - 새 주문 알림 (웹 푸시)
    - 재고 부족 알림
    - 환율 급변 알림

#### 19. 고객 문의 시스템 없음
- **현재 상황**: 푸터에 "고객센터"가 있지만 FAQ 더미 텍스트만
- **문제점**: 고객 불만 사항을 수집하고 대응할 방법이 없음
- **개선 방안**:
  - **1:1 문의 게시판**
    - 로그인 필수
    - 주문번호 선택 (해당 주문 관련 문의)
    - 카테고리 선택 (배송, 환불, 상품, 기타)
    - 관리자 답변 기능
  - **카카오톡 채널 연동**
    - 실시간 상담 버튼
    - 카카오톡으로 문의 연결
  - **챗봇 (간단한 FAQ)**
    - "배송은 얼마나 걸리나요?" → 자동 응답
    - "환불은 어떻게 하나요?" → 자동 응답
  - **이메일 지원**
    - support@ymarketer.kr
    - 자동 응답 메일 (접수 확인)

#### 20. 환불 정책이 명시되지 않음
- **현재 상황**: 구매대행 특성상 환불이 어려운데 정책이 없음
- **문제점**: 고객과 분쟁 발생 시 기준이 없음
- **개선 방안**:
  - **이용약관 작성**
    - 구매대행 프로세스 설명
    - 환불 가능 조건 (예: 상품 배송 전, 상품 하자)
    - 환불 불가 조건 (예: 고객 변심, 이미 배송 시작)
    - 환불 처리 기간 (영업일 기준 7-10일)
  - **FooterModal 실제 내용 작성**
  - **주문서에 환불 정책 링크 추가**
    - "환불 정책에 동의합니다" 체크박스

### 🟡 권장 (Medium Priority)

#### 21. 구독 서비스 기회
- **개선 방안**:
  - 월 정액제 멤버십
    - 예: 월 9,900원으로 무제한 무료 배송
    - 추가 할인 혜택 (5-10%)
    - 우선 배송
  - 정기 배송 상품
    - 뷰티 제품, 건강식품 등

#### 22. 제휴 마케팅 (Affiliate)
- **개선 방안**:
  - 추천인 코드 시스템
    - 각 회원에게 고유 추천 코드 발급
    - 추천한 사람이 구매 시 양쪽 모두에게 포인트 지급
  - 인플루언서 협업
    - 전용 할인 코드 제공
    - 판매 수수료 지급

#### 23. B2B 기회
- **개선 방안**:
